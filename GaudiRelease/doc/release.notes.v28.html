Gaudi v28r2 (2017-03-21)

Package Coordinators: Marco Clemencic, Charles Leggett, Benedikt Hegner

Externals version: LCG_88

Highlights

This minor release of Gaudi is mainly a bugfix release, nevertherless it features some new features, like

  • support for BINARY_TAG extensions for microarchitecture and optimizations (gaudi/Gaudi!289)
  • improvements to the graph scheduler
  • change of default optimization level to -O3
  • changed the signature of Algorithm::sysExecute (gaudi/Gaudi!273)

The full list of changes can be found on Gitlab.

Follows a list of JIRA tickets closed in this release.

Bug

  • [GAUDI-1280] - guess_heptools_version should not look into other platforms
  • [GAUDI-1287] - the Makefile wrapper around cmake does not respect the build tool used in the configuration
  • [GAUDI-1290] - inconsitency in the (Python) representation of configurables
  • [GAUDI-1293] - MessageSvc colors not consistent between Gaudi.exe and gaudirun.py

Task

  • [GAUDI-1279] - fix/clean up special test on status code checking (GaudiExamples)
  • [GAUDI-1288] - add dependency to GDB in Gaudi

Improvement

  • [GAUDI-1276] - Do we really need read handles inside Property?

Gaudi v28r0 (2016-11-08)

Package Coordinators: Marco Clemencic, Charles Leggett, Benedikt Hegner

Externals version: LCG_86

Highlights

This major release of Gaudi features several changes.

Together with usual minor fixes, improvements and clean ups, we have the following major changes:

  • dropped support for CMT builds
  • mimumum required C++ standard is now C++14
  • rewrite of components property classes
The full list of changes can be found on Gitlab.
